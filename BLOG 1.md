 Tipos de instâncias EC2
On-Demand: você paga pelo tempo de uso, sem compromisso de longo prazo.

Reserved Instances: você reserva capacidade por 1 ou 3 anos, com desconto.

Spot Instances: você aproveita capacidade ociosa da AWS com preços muito mais baixos, mas a instância pode ser encerrada se o preço subir ou a capacidade for necessária.

Dedicated Hosts: hardware exclusivo para você, útil em casos de licenciamento específico.

📌 Templates e Blueprints
Amazon Machine Images (AMIs): são “templates” de sistemas operacionais e softwares pré-configurados que você pode usar para criar instâncias rapidamente.

Blueprints: usados em serviços como Lightsail (mais simples que EC2), onde você escolhe um “blueprint” pronto (por exemplo, WordPress, LAMP, Node.js) e já sobe uma VM com tudo configurado.

Launch Templates: no EC2, você pode criar um template de lançamento que define configuração de instância (tipo, AMI, rede, storage, etc.) para reutilizar em múltiplos deployments.

🎯 Em resumo
EC2 = serviço de máquinas virtuais.

Spot = instâncias baratas, mas sem garantia de continuidade.

Templates/Blueprints = formas rápidas de subir instâncias já configuradas.

MOSTRE OS TIPOS DE EC2 DE MAQUINA
O Amazon EC2 oferece diferentes tipos de instâncias de máquinas virtuais, agrupadas em famílias que variam conforme o foco: uso geral, computação, memória, armazenamento, aceleração por GPU e HPC. A escolha depende da carga de trabalho que você precisa rodar. 

📌 Principais famílias de instâncias EC2
Família	Características	Exemplos de uso
Uso Geral (General Purpose)	Equilíbrio entre CPU, memória e rede	Servidores web, bancos de dados pequenos/médios, ambientes de desenvolvimento
Otimizadas para Computação (Compute Optimized)	Alta performance de CPU	Processamento em lote, transcodificação de mídia, servidores de jogos
Otimizadas para Memória (Memory Optimized)	Grande quantidade de RAM	Bancos de dados em memória, análises em tempo real, aplicações empresariais
Computação Acelerada (Accelerated Computing)	GPUs ou FPGAs para cálculos intensivos	Machine learning, renderização gráfica, simulações científicas
Otimizadas para Armazenamento (Storage Optimized)	Alto throughput de leitura/escrita	Bancos de dados de alto desempenho, big data, processamento de logs
HPC (High Performance Computing)	Máxima performance em escala	Simulações complexas, aprendizado profundo, renderização de efeitos visuais


🔎 Convenção de nomes
Cada instância tem um nome que indica família e tamanho.

Exemplo: t2.micro

t2 → família de uso geral.

micro → tamanho pequeno (poucos vCPUs e RAM).

Outro exemplo: c5.large

c5 → família otimizada para computação.

large → tamanho médio.

🎯 Como escolher
Se precisa de equilíbrio: comece com t3.micro ou t3.medium (uso geral).

Se precisa de CPU forte: escolha c5.large ou c6i.xlarge.

Se precisa de muita memória: vá de r5.large ou x1e.32xlarge.

Se precisa de GPU: use p3.2xlarge ou g4dn.xlarge.

Se precisa de armazenamento intenso: opte por i3.large ou d2.xlarge.

⚠️ Pontos de atenção
Spot Instances: podem ser muito mais baratas, mas não garantem disponibilidade contínua.

Reserved Instances: reduzem custo se você se comprometer por 1–3 anos.

Launch Templates: permitem criar configurações padronizadas para instâncias, facilitando replicação.

👉 Em resumo: o EC2 oferece famílias de instâncias para diferentes cenários — desde servidores simples até HPC com GPUs. A escolha depende do equilíbrio entre custo, desempenho e requisitos da aplicação.
